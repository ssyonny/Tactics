## 2. 개요
- 게임 개요(한 줄): 탑다운 시점의 액션 프로토타입. AI 동료들과 적을 물리치며 진행하는 싱글플레이 게임.
- 장르: 액션 / 탑다운
- 목표 플랫폼: 
  - Windows (Win64, Editor) - Phase 1
  - Android (9.0+ / API 28+) - Phase 2
  - iOS (13.0+) - Phase 2
- 타깃 유저: 13세 이상, 캐주얼에서 인디 게임을 즐기는 플레이어 (PC 및 모바일)
- 핵심 컨셉 및 차별점: 짧은 플레이 루프(3~7분), 간단한 업그레이드 시스템, 플레이 중심의 빠른 전투, 크로스 플랫폼 지원
- KPI(성공 지표): 
  - 평균 세션 시간 >= 5분
  - 플레이어 유지율(DAU/MAU) 베타 기준 10%
  - 모바일 크래시율 < 1%
  - 모바일 평균 FPS >= 30

## 3. 범위(스코프)
- MVP(필수):
  - 플레이어 이동 및 기본 공격, AI 동료의 배치와 명령(공격, 후퇴, 돌격)
  - 3개의 레벨(맵)과 레벨 전환
  - 기본 적(순찰, 추적, 체력/데미지)
  - 기본 UI(체력바, 레벨 타이틀, 간단한 메뉴)
  - .uproject 파일과 Source 코드(플랫폼: Win64) 저장소 반영
- 향후(선택):
  - 적 타입 추가, 무기/아이템 시스템

## 4. 용어집
프로젝트 전반에 사용되는 주요 용어를 표준화합니다.

- **캐릭터(Character)**: 플레이어가 조종하거나 AI가 제어하는 게임 내 유닛
- **동료/컴패니언(Companion)**: 플레이어를 지원하는 AI 제어 캐릭터
- **AP(Action Point, 행동포인트)**: 본거지에서 활동하여 획득하는 포인트. 이벤트 발생 간격을 단축시킴
- **HP(Hit Point)**: 체력
- **본거지(Base)**: 자원 채집 및 생산 활동을 하는 메인 허브 레벨
- **미션(Mission)**: 전투 목표가 있는 레벨 (레벨 A~E)
- **MVP(Minimum Viable Product)**: 최소 실행 가능 제품 — 프로토타입의 필수 기능 범위

## 5. 게임 플레이(기능 요구사항)
- 작성 규칙: 각 항목에 설명, 우선순위(필수/선택), 수용 기준(테스트 방법)을 적음.

### 5.1 플레이어 이동
- 설명: WASD 또는 왼쪽 조이스틱으로 8방향 이동, 마우스 방향으로 조준(또는 조이스틱 조준)
- 우선순위: 필수
- 수용 기준: WASD 입력으로 캐릭터가 이동하고 애니메이션(Idle/Run) 전환이 부드럽게 이뤄짐. 입력 지연 < 100ms.

### 5.2 공격
- 설명: 마우스 클릭 또는 버튼으로 공격. 근접/원거리 공격타입 전환.
- 우선순위: 필수
- 수용 기준: 공격 입력 시 공격 애니메이션 재생, 적과 충돌 시 권장 데미지 공식(DamageTaken = round(BaseDamage * 100/(100+Armor)))에 따라 데미지 적용, 데미지 수치가 UI에 반영됨.

### 5.3 적 AI
- 설명: 기본 적은 순찰하다가 플레이어를 시야 내에서 감지하면 추적 및 공격.
- 우선순위: 필수
- 수용 기준: 감지 거리 내 플레이어를 추적, 전투 종료 후 적 소멸 및 점수 반영.

### 5.4 레벨/맵
- 설명: 3개의 독립된 레벨, 레벨 간 전환 시스템
- 우선순위: 필수
- 수용 기준: 레벨 전환 시 로드 시간 < 5초(로컬), 플레이어 포지션 초기화 또는 유지(디자인 결정에 따름).

### 5.5 UI/UX
- 설명: 체력바, 레벨명, 일시정지 메뉴, 간단한 설정
- 우선순위: 필수
- 수용 기준: 모든 UI 요소는 해상도 변경(예: 1280x720 → 1920x1080)에서 잘 배치되어 가독성 유지.

### 5.6 세이브/로드
- 설명: 플레이 진행(레벨 완료 상태)만 간단 저장
- 우선순위: 선택
- 수용 기준: 게임 종료 후 재접속 시 마지막 체크포인트를 불러올 수 있음.

### 5.7 신규 미션 및 맵 업데이트 기능
- 설명: 런타임에 신규 맵 에셋을 동적으로 다운로드하고 게임에 통합
- 우선순위: 선택 (Phase 3: 라이브 서비스)
- 기술 요구사항:
  - 맵 버전 관리 시스템 (서버에서 버전 체크)

  - 에셋 패키징: `.pak` 파일 형태로 배포
  - 다운로드 진행률 표시 (프로그레스바)
  - 다운로드 실패 시 재시도 로직

## 6. 게임 진행 및 플레이 방법

목적: 플레이어가 게임을 어떻게 경험하고 어떤 방식으로 진행하는지, 각 단계에서 기대되는 동작과 성공 기준을 정의한다.

### 6.1 게임 루프 (Game Loop)
- 설명: 플레이어가 한 세션 동안 반복하는 기본 루프.

#### 6.1.1 본거지 탐험
- 플레이어는 본거지에서 자원을 채취하고 채취한 자원으로 생산활동을 한다.
  집도 짓고 식량을 생산하고 무기와 소모품을 만들 수 있다. 자원의 획득 방법은 채집과 수렵이 있다.
- 우선순위: 필수
- 자원 시스템:
  - 4종 자원: 나무, 광석, 고기, 열매
  - 각 자원 기본 상한: 100 (업그레이드로 증가 가능)
  - 채집: 나무(5초/개), 광석(10초/개), 열매(3초/개)
  - 수렵: 고기(사냥 성공 시 10~20개, 30초 소요)
- AP(행동포인트) 획득: 자원 1개당 AP +1
- 이벤트 발생 시 미션 플레이로 이동하고 3분 이내에 동료를 배치해 전투를 대비해야 한다.
- 수용 기준: 자원 상한 100 확인, AP 누적 표시, 이벤트 트리거 정상 동작
- 단일 루프(레벨 시작 → 목표 달성 → 보상 적용)가 평균 3~7분 소요(디자인 목표).
- 검증 방법: 플레이 세션 리플레이 로그 또는 수동 플레이 타임 측정(10회 평균).

#### 자원 시스템 상세 (테이블)

| 자원 | 획득 방법 | 소요 시간 | 획득량 | AP 보너스 | 용도 예시 |
|------|----------|---------|--------|-----------|----------|
| 나무 | 채집 | 5초 | 1개 | +1 | 건물 건설, 무기 제작 |
| 광석 | 채집 | 10초 | 1개 | +1 | 무기/방어구 제작 |
| 열매 | 채집 | 3초 | 1개 | +1 | 식량, 소모품 |
| 고기 | 수렵 | 30초 | 10~20개 | +10~20 | 식량, 고급 소모품 |

**자원 소비 예시 (초기 설정)**
- 집 건설: 나무 20, 광석 10
- 기본 무기: 나무 5, 광석 3
- 소모품(회복약): 열매 3, 고기 1

#### 6.1.2 미션 플레이 (AI기반 자동 플레이)
- 미션 플레이로 진입 시 본거지 탐험에서 수행한 내용을 저장하고 이 레벨로 이동해야한다.
- 이벤트 발생 조건:
  - 기본 간격: 300초 (5분)
  - AP가 100 초과 시: 초과한 AP 1포인트당 5초씩 간격 단축
  - 공식: EffectiveInterval = max(60초, 300초 - (AP - 100) * 5초)
  - 예시: AP=100 → 300초, AP=110 → 250초, AP=148+ → 60초(최소)
- 이벤트가 발생하여 미션 플레이로 이동하면 승리 목표를 미리 알려주고 파티를 선택하고 그중에 플레이할 캐릭터를 선택할 수 있다. 각 캐릭터는 캐릭터의 고유 특성에 따라 무기를 선택할 수 있다. 이 시간은 준비 시간으로 1분이 주어진다. 단, 전투 시작을 누르면 이 시간과 관계없이 게임이 시작한다.
    - 미션의 종류는 다음과 같다. 요인암살(레벨 C), 요인 구출(레벨 D), 모든 적 섬멸(레벨 A), 탈출(레벨 E), 생존(레벨 B) 이다. 
    각 미션마다 맵과 성공 조건이 있다.
    - 전투의 종료는 성공목표로 목표의 달성, 실패 목표로 캐릭터의 사망이 있다. 플레이어는 캐릭터를 선택하여 조종할 수 있는데 조종하는 캐릭터가 사망하면 실패한다. AI로 움직이는 다른 동료의 죽음은 전투의 승패와 관계 없다.
    전투 종료 시 전투에 참여했던 동료들의 레벨업, 무기 숙련도는 사용율에 따라 증가 한다. 사망한 캐릭터는 난이도가 상급일 때 능력치 하락 등의 패널티가 발생한다. 
  
### 6.2 목표(목적) 및 성공 조건
- 설명: 각 레벨의 주요 목표 유형(예: 전멸, 생존, 오브젝트 수집, 보스 처치).
- 우선순위: 필수
- 예시:
  - 레벨 A: 모든 적 섬멸 → 성공 조건: 남은 적 수 = 0
  - 레벨 B: 2분 생존 → 성공 조건: 생존 타이머 >= 120s
  - 레벨 C: 요인 암살 -> 성공조건 : 지정 타겟의 수 = 0
  - 레벨 D: 요인 구출 -> 성공조건 : 남은 적 수 = 0 또는 안전지역까지 케릭터 이동. 이때 구출되는 인원은 5미터 이내에 조종하는 케릭터를 자동으로 따라와야한다.
  - 레벨 E: 탈출 -> 성공 조건 : 플레이어가 조종하는 케릭터가 주어진 구역으로 이동.(케릭터 변경 불가)
- 수용 기준:
  - 성공/실패 판정은 즉시 UI에 표시되고, 리워드 또는 재시작 팝업이 나온다(지연 < 1s).
- 검증 방법: 자동화된 시뮬레이터 또는 수동 플레이에서 판정 표시 타임 측정.

#### 미션 타입 상세 (테이블)

| 레벨 ID | 미션 타입 | 성공 조건 | 실패 조건 | 예상 플레이 시간 | 맵 특성 |
|---------|----------|----------|----------|----------------|---------|
| 레벨 A | 전멸 | 남은 적 수 = 0 | 플레이어 캐릭터 사망 | 5~7분 | 중형 맵, 적 15~20 |
| 레벨 B | 생존 | 생존 타이머 >= 120s | 플레이어 캐릭터 사망 | 2분+ | 소형 맵, 적 연속 스폰 |
| 레벨 C | 요인 암살 | 지정 타겟 사망 | 플레이어 캐릭터 사망 | 4~6분 | 중형 맵, 보스 1 + 적 10 |
| 레벨 D | 요인 구출 | 구출 대상을 안전지역으로 이동 | 플레이어/구출 대상 사망 | 5~8분 | 대형 맵, 적 20~30 |
| 레벨 E | 탈출 | 플레이어가 탈출 지역 도달 | 플레이어 캐릭터 사망 | 3~5분 | 중형 맵, 적 10~15 |

**추가 규칙**
- 레벨 D: 구출 대상은 플레이어 캐릭터 5m 이내에서 자동 추종
- 레벨 E: 전투 중 캐릭터 변경 불가

### 6.3 레벨/스테이지 진행
- 설명: 레벨 간 전환 방식과 저장/로드 동작
- 우선순위: 필수
- 요구사항:
  - 레벨 끝나면 결과 화면(획득 보상, 통계)을 표시하고 다음 레벨로 이동 또는 종료 선택지를 제공.
  - 레벨 전환 시 로드 시간 < 5s(개발 환경 기준).
  - 캐릭터 변경 시 시점 변경 후 캐릭터 조종(5초 쿨다운 적용)
- 수용 기준:
  - 결과 화면 버튼(다음/메인) 동작 지연 < 200ms.
- 검증 방법: UI 스크립트 테스트 및 로드 시간 자동 측정.

### 6.4 플레이어 행동(액션)과 입력 맵
- 설명: 플레이어가 수행할 수 있는 주요 액션 및 입력 맵핑
- 필수 액션 목록(우선순위: 필수):
  - 이동: WASD / 좌스틱
  - 조준: 마우스 커서 / 우스틱
  - 기본 공격: 왼쪽 클릭 / 버튼 X
  - 스킬/특수: Q,E 또는 버튼 Y/B
  - 상호작용(아이템 획득/문 열기): F / 버튼 A
  - 일시정지/메뉴: Esc / Start
- 수용 기준:
  - 모든 입력은 100ms 미만의 응답성(개발 환경 평균)으로 동작.
  - 컨트롤 리맵핑이 설정 화면에서 즉시 저장되어 게임 내에서 반영.
- 검증 방법: 입력 지연 측정, 리맵핑한 설정으로 10분 플레이 후 반영 여부 확인.

### 6.5 난이도 및 밸런싱
- 설명: 본거지 플레이일대 난이도 선택가능, 선택된 난이도에 따라 보상과 패널티가 다름.
- 우선순위: 선택(초기 버전은 고정 난이도 "Normal" 제공)
- 요구사항:
  - 난이도 레벨: Easy / Normal / Hard (초기: Normal 기본)
  - 적 스폰 수, HP, 데미지 등 변수는 난이도 별로 가중치를 적용하여 조정 가능해야 함.
- 수용 기준:
  - Normal에서 평균 플레이 시간(레벨 당) 및 사망률(레벨 당)을 수집하여 이후 밸런싱 목표 수치 설정.
- 검증 방법: 플레이 데이터 수집 및 로그 분석(초기 베타 100세션 권장).

#### 난이도 가중치 (테이블)

| 난이도 | 적 HP | 적 데미지 | 적 스폰 수 | 보상 배율 | 캐릭터 사망 페널티 |
|--------|-------|----------|-----------|----------|------------------|
| Easy | -30% | -20% | -20% | 0.8x | 없음 |
| Normal | 100% | 100% | 100% | 1.0x | 경험치 -10% |
| Hard | +50% | +30% | +30% | 1.5x | 경험치 -20%, 능력치 일시 감소 |

**추가 설명**
- 난이도는 본거지에서만 변경 가능 (미션 중 변경 불가)
- Hard 난이도 능력치 감소: 공격력 -5%, 방어력 -5% (다음 레벨업까지)

### 6.6 보상 시스템(리워드)
- 설명: 플레이어가 목표를 달성하거나 적을 처치했을 때 주어지는 보상 설계
- 구성 요소:
  - 즉시 보상: 골드, 아이템 드롭
  - 진행 보상: 레벨 완료 보너스, 경험치(레벨 업)
  - 상점/업그레이드(선택): 획득 자원으로 스탯/무기 업그레이드
- 우선순위: 필수(기본 보상), 선택(상점/업그레이드)
- 수용 기준:
  - 레벨 보상은 즉시 인벤토리에 반영되어야 하며, UI에 획득 로그가 표시된다(지연 < 500ms).
- 검증 방법: 자동화 스크립트로 보상 획득 시 인벤토리 상태 변화 확인.

### 6.7 진행 추적(세이브 / 체크포인트)
- 설명: 세이브 포인트와 자동 저장 정책
- 요구사항:
  - 레벨 완료 또는 주요 체크포인트에서 자동 저장 수행.
  - 수동 저장(메뉴를 통한) 옵션 제공(선택).
- 수용 기준:
  - 자동 저장 성공 시 백업 파일(타임스탬프 포함) 생성, 로드 시 마지막 체크포인트에서 재개 가능.
- 검증 방법: 저장 파일 생성 여부 확인 및 로드 검증(3회 반복).

### 6.8 튜토리얼 / 온보딩
- 설명: 신규 플레이어를 위한 기본 튜토리얼 흐름
- 요구사항:
  - 튜토리얼 레벨 또는 인터랙티브 팝업(첫 플레이 시)로 기본 조작(이동/공격/상호작용)을 소개.
  - 튜토리얼은 스킵 가능하며, 재시청 가능해야 함.
- 수용 기준:
  - 신규 플레이어가 튜토리얼 완료 후 3분 이내에 기본 전투 루프(이동+공격)를 성공적으로 수행할 확률 ≥ 90% (초기 베타 측정).
- 검증 방법: 신규 사용자 그룹(5–10명) 대상으로 플레이테스트 결과 수집.

### 6.9 메트릭 및 로깅 (운영 목적)
- 설명: 게임 진행을 모니터링하기 위한 기본 이벤트와 메트릭
- 권장 이벤트:
  - SessionStart, LevelStart(levelId), LevelComplete(levelId, time), PlayerDeath(levelId), ItemAcquired(itemId), TutorialCompleted
- 우선순위: 필수(기본 이벤트)
- 수용 기준:
  - 각 이벤트는 플레이어 행동 발생 후 5s 이내에 로그로 기록되며, 서버 전송(있는 경우)은 비동기로 처리.
- 검증 방법: 로그 파일/시스템에서 이벤트 발생과 기록 타임스탬프 확인.

### 6.10 수용 기준 요약(체크리스트)
- 플레이어는 10초 이내에 레벨 목표를 식별할 수 있다.
- 레벨 전환 로드 시간 < 5s (개발 환경 기준).
- 입력 응답성 평균 < 100ms.
- 결과/보상 UI 반영 지연 < 500ms.
- 자동 저장 후 마지막 체크포인트에서 재개 가능.

### 권장 설계 결정(확정안)
프로토타입에서 반복/오해를 줄이고 개발 착수를 빠르게 하기 위해 아래 디자인 결정을 권장합니다. 팀 내 합의 후 이 문서의 정식 요구사항으로 확정하세요.

1) 데미지 공식 (확정 권장)
 - 공식: DamageTaken = round(BaseDamage * (100 / (100 + Armor)))
 - 설명: Armor가 증가할수록 데미지 감소 효과는 비선형적으로 적용됩니다. K 값(여기서는 100)은 튜닝 파라미터이며 초기값으로 사용합니다.
 - 예시:
   - BaseDamage=50, Armor=0 → DamageTaken = 50
   - BaseDamage=50, Armor=25 → DamageTaken = 40
   - BaseDamage=50, Armor=100 → DamageTaken = 25
 - 수용 기준: 단위 테스트 5개(Armor: 0, 25, 50, 100, 500) 통과. 데미지 값은 라운드 처리되어 UI와 일치.

2) 컴패니언(동료) 사망/부활 규칙 (MVP) - **확정** (2025-11-07)
 - 규칙: 
   - 전투 중 사망 시 "쓰러짐" 상태로 전환 (시각적 표시: 누워있는 애니메이션)
   - 전투 종료 후 즉시 자동 부활 (애니메이션 없음, 결과 화면 전)
   - 부활 시 HP = MaxHP * 50%
   - 패널티: 난이도별 차등 적용
     - Easy: 패널티 없음, HP 100% 부활
     - Normal: 해당 전투 획득 경험치 -10%, HP 50% 부활
     - Hard: 해당 전투 획득 경험치 -20%, HP 30% 부활, 공격력/방어력 -5% (다음 레벨업까지)
 - 이유: 프로토타입 단계에서 영구사망은 UX와 복잡도를 높여 테스트가 어려워짐. 난이도별 차등 패널티로 전술적 깊이 제공.
 - 수용 기준: 
   - 전투 중 동료가 HP 0이 되면 상태는 'downed'로 표시되고 쓰러진 애니메이션 재생
   - 전투 종료 시 상태가 'alive'로 변경되며 난이도별 HP/패널티 적용
   - Hard 난이도에서 능력치 감소는 다음 레벨업 시 복구
 - 상세 문서: `docs/COMPANION_SYSTEM.md` 참조

3) 레벨 전환 시 플레이어 포지션 규칙
 - 기본 규칙: 레벨 전환 시 플레이어는 해당 레벨의 지정된 스폰 포인트로 초기화.
 - 예외: 디자인 필요 시 '포지션 유지' 플래그(맵 속성)로 예외 허용.
 - 수용 기준: 레벨 전환 후 플레이어 위치가 스폰 지점 좌표와 일치.

4) 자동 저장/세이브 스키마 (JSON, 초기)
 - 저장 시점: 레벨 완료, 주요 체크포인트, 메뉴 수동 저장.
 - 기본 저장 항목: 플레이어(id, level, hp, maxHp, exp, position), party(멤버 id/level/hp/status), inventory(itemId/count), completedLevels[], timestamp, version
 - 무결성: 저장 시 JSON 문자열에 대해 SHA256 체크섬을 생성하여 `checksum` 필드에 포함. 로드 시 체크섬 검증 실패 시 경고 및 복구 플로우.
 - 예시(save_example):
 {
   "version":"0.1",
   "timestamp":"2025-11-07T12:34:56Z",
   "player":{
     "id":"Player01",
     "level":3,
     "hp":82,
     "maxHp":100,
     "position":{"x":123.4,"y":45.6,"z":0.0},
     "exp":420
   },
   "party":[
     {"id":"CompA","level":2,"hp":50,"status":"alive"},
     {"id":"CompB","level":1,"hp":0,"status":"dead"}
   ],
   "inventory":[{"itemId":"gold","count":150},{"itemId":"potion_small","count":2}],
   "completedLevels":["Level_01","Level_02"],
   "checksum":"<sha256-of-json-without-checksum>"
 }
 - 수용 기준: 저장 후 파일 생성 및 체크섬 검증 통과, 동일 데이터 로드 시 필드 값 일치.

5) 이벤트 / 행동포인트(AP) 공식
 - 기본값: BaseInterval = 300s (5분)
 - AP 적용: EffectiveInterval = max(MinInterval, BaseInterval - (AP - 100) * IntervalReduction)
   - IntervalReduction = 5s (AP 100 초과분에 대해)
   - MinInterval = 60s
 - 예시: AP=100 → 300s, AP=110 → 250s, AP=148+ → 60s(최소 클램프)
 - 수용 기준: 시뮬레이터로 AP 별 평균 트리거 간격 측정(10회) 결과가 계산값과 일치.

6) 엔진 버전 고정
 - 권장: Unreal Engine 5.1.x(예: 5.1.2)로 패치 버전 고정. 프로젝트 설정과 README에 엔진 버전 명시.
 - 이유: 빌드 재현성과 플러그인/에디터 호환성 보장.

7) 수용 기준/테스트 요약(요청된 항목 반영)
 - 이동 응답성: 자동 입력 10회 평균 응답 < 100ms
 - 레벨 로드: 개발 PC에서 3회 평균 로드 시간 < 5s
 - 데미지: 데미지 단위 테스트(5개 케이스) 통과
 - 자동 저장: 저장 파일 생성 + 체크섬 검증 통과

    - Easy: 패널티 없음, HP 100% 부활
    - Normal: 해당 전투 경험치 -10%, HP 50% 부활
    - Hard: 해당 전투 경험치 -20%, HP 30% 부활, 능력치 -5% (다음 레벨업까지)
  - 수용 기준: 전투 중 동료가 downed 상태가 되면 전투 종료 시 alive로 복구되고 난이도별 HP/패널티 적용됨. 상세 문서: `docs/COMPANION_SYSTEM.md`

- 레벨 전환 포지션
  - 결정: 기본적으로 스폰 포인트로 초기화. 필요 시 맵 속성으로 '포지션 유지' 옵션 허용.
  - 수용 기준: 레벨 전환 후 플레이어 위치가 스폰 좌표와 일치(예외 플래그 적용 시 별도 동작).

- 세이브 포맷
  - 결정: JSON 포맷 + SHA256 체크섬, 저장 시점: 레벨 완료/체크포인트/수동
  - 수용 기준: 저장파일 생성 및 체크섬 검사 통과, 로드 시 데이터 일관성 보장.

- 이벤트/AP 공식
  - 결정: EffectiveInterval = max(60s, 300s - (AP - 100) * 5s)
  - 수용 기준: 시뮬레이터 테스트 10회 평균 트리거 간격이 계산값과 유사.
